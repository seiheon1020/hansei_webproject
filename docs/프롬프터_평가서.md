## 2. 강점 분석

### ✅ 2.1 빠른 피드백과 방향 수정

- 결과물이 기대와 다를 때 즉시 Undo하고 명확한 재지시 제공
- "UI는 유지하고 로직만 수정해줘"와 같은 교정 프롬프트 활용
- 불필요한 지연 없이 다음 단계로 진행

### ✅ 2.2 점진적 개선 접근

- 한 번에 모든 것을 요구하지 않고 기능 단위로 요청
- 작은 변경 → 확인 → 다음 요청의 반복 사이클 유지
- 이는 AI 협업에서 매우 효과적인 전략

### ✅ 2.3 구체적인 정렬 규칙 제시

```
정렬 규칙:
1) 현재 진행 중인 대회
2) 가장 가까운 미래 대회
3) 이미 종료된 과거 대회는 가장 아래
```
- 위와 같이 번호와 조건을 명시한 요청은 모범적인 프롬프트

## 3. 개선이 필요한 영역

### ⚠️ 3.1 초기 요구사항의 모호성 (심각도: 중)

**문제 상황:**
- 초반 "캘린더 자동 생성해줘" 요청은 범위가 불명확
- AI가 과도하게 해석하여 UI까지 변경하는 결과 초래
- 이후 "이전 모습으로 바꿔줘" 교정 필요

**개선 방안:**
```
❌ "캘린더 수정해줘"
✅ "캘린더의 렌더링 로직에서 이벤트 표시 조건만 수정해줘.
   현재: startDate~endDate 범위면 표시
   변경: 해당 날짜에 실제 경기(matches)가 있을 때만 표시
   UI 레이아웃은 그대로 유지"
```

### ⚠️ 3.2 컨텍스트 전달 부족 (심각도: 중)

**문제 상황:**
- 여러 번의 Undo로 인해 현재 코드 상태가 불명확해짐
- AI가 이전 버전 기준으로 패치를 시도하여 실패하는 경우 발생

**개선 방안:**
- 중요한 Undo 후에는 "현재 코드 상태를 확인해줘" 요청
- 또는 변경된 파일의 핵심 부분을 첨부

### ⚠️ 3.3 기술 용어 혼용 (심각도: 하)

**문제 상황:**
- "대회"와 "경기"를 혼용하여 사용
- tournament vs match의 구분이 불명확한 요청 존재

**개선 방안:**
- 프로젝트 초반에 용어 정의 명시
- `tournament`: 대회 전체 (LCK Winter Split)
- `match`: 개별 경기 (T1 vs Gen.G)

### ⚠️ 3.4 검증 단계 생략 (심각도: 중)

**문제 상황:**
- 변경 후 실제 브라우저에서 테스트했는지 확인 어려움
- AI에게 "확인해줘"라고 요청했으나 실제 렌더링 검증은 불가

**개선 방안:**
- 각 기능 완료 후 직접 브라우저 테스트 수행
- 발견된 이슈를 구체적으로 전달

## 5. AI 협업 역량 발전을 위한 제언

### 5.1 단기 개선 (즉시 적용 가능)

1. **요청 전 스스로 질문하기**
   - "이 요청으로 AI가 무엇을 변경할까?"
   - "변경 범위가 명확한가?"
   - "예상치 못한 부작용은?"

2. **제약 조건 명시 습관화**
   ```
   요청: OOO 기능 추가
   제약: 
   - 기존 UI 레이아웃 유지
   - XX 파일만 수정
   - YY 함수는 건드리지 않음
   ```

3. **결과 확인 체크리스트**
   - [ ] 브라우저에서 테스트
   - [ ] 다른 페이지에 영향 없는지 확인
   - [ ] 에러 콘솔 확인

### 5.2 중기 개선 (프로젝트 단위)

1. **용어집 작성**: 프로젝트 시작 시 핵심 용어 정의
2. **스펙 문서 활용**: 주요 기능은 미리 명세 작성 후 AI에게 전달
3. **버전 관리 습관**: Git 커밋으로 안정적인 체크포인트 생성

### 5.3 장기 개선 (역량 개발)

1. **프롬프트 엔지니어링 학습**: 체계적인 프롬프트 작성 기법 습득
2. **기술 이해도 향상**: AI가 생성한 코드를 읽고 이해하는 능력
3. **협업 패턴 정립**: 자신만의 AI 협업 워크플로우 개발

## 6. 최종 평가 요약

### 잘한 점
- 빠른 피드백 루프 유지
- 점진적 개선 접근법
- 구체적인 정렬 규칙 제시

### 개선할 점
- 초기 요구사항 명확화
- 변경 범위(UI vs 로직) 명시
- 용어 일관성 유지
- 직접 테스트 후 피드백

### 성장 가능성
현재 B+ 수준의 프롬프터 역량을 보유하고 있으며, 위 개선점을 적용할 경우 **A 등급 도달 가능**. 특히 "제약 조건 명시" 습관 하나만으로도 협업 효율이 크게 향상될 것으로 예상됨.
